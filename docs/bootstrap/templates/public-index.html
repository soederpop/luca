Start Here

Everything below runs from your terminal in this project folder. The luca CLI discovers your commands/, features/, and endpoints/ automatically.

luca                      # list every available command
luca about                # project info + discovered helpers
luca describe features    # index of everything the container provides
luca describe fs          # full docs for any helper

Run Scripts luca run

Run any TypeScript file with the full container in scope — no imports of fs or path needed, the container provides everything.

luca run script.ts        # run a script with the container injected
luca eval "container.features.available"   # one-liners, great for prototyping
  • Files: container.feature('fs')
  • Paths: container.paths.resolve()
  • Utils: container.utils — uuid, lodash, string helpers

Run Markdown runnable docs

Markdown files with code blocks are executable. Your project ships with RUNME.md — an interactive demo you can run right now.

luca RUNME                # run the bundled interactive demo
luca run docs/some-guide.md   # any markdown with code blocks works

Write docs that never go stale — if the code blocks break, running the doc tells you.

Scaffold luca scaffold

Generate working boilerplate for every helper type. Each scaffold has a built-in tutorial.

command

A CLI command in commands/, runnable as luca <name>.

luca scaffold command deploy

feature

A container feature in features/ — reusable business logic.

luca scaffold feature cache

endpoint

A REST route in endpoints/, served by luca serve.

luca scaffold endpoint users

client / server

API clients and long-running servers.

luca scaffold client github

Add --tutorial to any scaffold for the full guide: luca scaffold feature --tutorial

Serve luca serve

You're looking at it. luca serve serves this public/ folder plus every file in endpoints/ as an API route.

luca serve                # static files from public/ + API from endpoints/

Try the bundled health endpoint: /api/health

  • Each file in endpoints/ maps to a route — endpoints/users.ts → /api/users
  • Export get, post, put functions; for DELETE use export { del as delete }
  • Share state across endpoints via ctx.request.app.locals

Assistants AI-native

Luca is built for conversational apps. Assistants combine a system prompt, tools (backed by container helpers), and a chat loop — scaffold one and talk to it immediately.

luca scaffold assistant helper   # generate an assistant
luca chat                        # interactive chat session
  • Tools are defined with Zod schemas — assistant.addTool() handles JSON Schema conversion
  • Assistants can call any container feature: files, HTTP, databases, your custom features
  • Run luca describe assistant for the full API

Content DB docs/ folder

Your docs/ folder is a queryable database of markdown documents. Schemas live in docs/models.ts; frontmatter is validated against them.

// in any script or command — container.docs is the contentDb feature
const docs = container.docs
const posts = await docs.query(docs.models.Post).fetchAll()
  • Markdown files with frontmatter become typed, queryable records
  • Define models with Zod in docs/models.ts — one model per subfolder
  • Run luca describe contentDb for query syntax and the full API
