# The Self Improvement Loop

I Believe I've reached a point, layed down enough features, enough documentation,
that we can get the project owner assistant to run a single task to "move the ball forward" and produce a plan for the next iteration loop, as well as a plea for why that is the most important thing to do next. For this reason I think it is a good idea to have a sort of North Star document that lays down core vision and principles, but is not ultra specific and allows for freedom.  The Assistant should treat this north star as immutable god's law when it comes to deciding how, what, and why to build.

A task loop itself will be anything it can accomplish with a luca script, including managing claude code sessions.  

It needs to do so, and produce a HANDOFF.md for the next iteration loop.

It needs to know that this HANDOFF.md will be reviewed by me the creator and possibly rejected, scrapping all of their hard work.
